Mirror_tp  mirrorType,
std::vector< unsigned int >  fixedStates 
)

Add mirror and state constraints to the first and last nodes on an arcset.

A mirror constraint is constructed based on the mirrorType specified in the input arguments. Additionally, a state constraint is constructed and applied to only the first node based on the fixedStates specified in the input arguments. Any pre-existing constraints on the initial and final nodes are deleted

Parameters
pArcpointer to the arcset
mirrorTypeDescribes how the trajectory is mirrored at both the initial and final nodes.
fixedStatesThe indices of any states at the initial node that should be fixed

Transition a arcset from the EM system to the SE system.

The relative orientation between the two systems at time t = 0 is described by the three angles thetaE0, thetaM0, and gamma. Accordingly, the EM arcset should have node epochs such that t = 0 corresponds to the desired geometry; adjusting the epoch for the entire set may be accomplished via the updateEpochs() function.

Parameters
EMNodesa CR3BP Earth-Moon arcset
pSESysa Sun-Earth CR3BP system data object
thetaE0the angle (radians) between the Sun-Earth line and the inertial x-axis at time t = 0.
thetaM0the angle (radians) between the Earth-Moon line and lunar "periapse" at time t = 0.
gammathe inclination (radians) of the lunar orbital plane relative to the ecliptic; this value is held constant.

Construct a full periodic orbit (PO) from a half-period representation of the orbit.

The half-period arc must start and end at mirror conditions. This function leverages the symmetry properties of the CR3BP to construct the second half of the full PO without numerical integration.

Parameters
halfPerArcAn arc that represents exactly half of the full periodic orbit
pFullPOA pointer to an object that stores the full periodic orbit
mirrorTpDescribes how the orbit is symmetric
Exceptions
Exceptionif either arcset pointer is null

std::vector< double >  ic,

Mirror_tp  mirrorType,

Generate an arcset that is a suitable guess for a symmetric periodic orbit in the CR3BP.

The output half-period arcset can be passed to cr3bp_correctHalfPerSymPO for corrections.

Note: This method ignores all crash events, so it is possible to compute a periodic orbit that passes through a primary.

Parameters
pSysthe dynamical system
icnon-dimensional initial state vector
halfPeriodnon-dimensional orbit period
halfPerNumNodesthe number of nodes to use for HALF of the periodic orbit; more nodes may result in a more robust correction
orderthe number of revolutions about the system/primary this orbit completes before it repeats periodically. Think of a period-3 DRO (order = 3) or a butterfly (order = 2)
mirrorTypehow this periodic orbit mirrors in the CR3BP
fixedStatesa vector containing the indices of which initial states we would like to fix. Not all states are possible for each mirror condition. See the enum definition for specific details.
toltolerance to use in the corrections process
Returns
An arcset that represents half of the desired periodic orbit and is constrained according to the specified mirror conditions
Exceptions
Exceptionif mirrorType is invalid
DivergeExceptionif the multiple shooting algorithm cannot converge on a mirrored solution.

Transform CR3BP rotating coordinates to inertial, dimensional coordinates.

Parameters

epoch0Epoch associated with t = 0 in the CR3BP; epoch in seconds past J2000 (i.e., ephemeris time)
centerIxThe index of the primary that will be the center of the inertial frame. For example, if an Earth-Moon CR3BP trajectory is input, selecting centerIx = 1 will produce Earth-centered inertial coordinates and selecting centerIx = 2 will produce Moon-centered inertial coordinates. A choice of centerIx = 0 produces system barycenter-centered inertial coordinates.
