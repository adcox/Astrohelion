Detailed Description

A data structure to represent a family of orbits.

Currently, this structure assumes family members are periodic.

std::vector< double > *  dataSet,
Constraint  matchCon 
) const
protected

Locate a family member with a specific attribute.

This function locates a family member or set of members that have a specific value for one of the variables of interest (e.g. coordinates, Jacobi, TOF). Exact matches and interpolated matches are returned; interpolated matches are computed using a differential corrections algorithm.

Parameters
valuethe value the family member should have
dataSeta pointer to a vector containing the set of values to search for matches in. For example, if the value I pass in contains a specific TOF, then dataSet points to a vector containing the TOFs for the entire family, sorted according to this family's sortType.
matchCona constraint that can be applied in a corrections scheme that will ensure the corrected trajectory has the desired value for the variable of interest.
Returns
a vector of matches. If no matches are returned, the vector will be empty.

◆ sortMembers()

void astrohelion::Fam_cr3bp::sortMembers ( )

Sort the family members by the specified sort variable (in ascending order)

The sorting variable is specified by sortType; this is the variable that best describes the natural progression of the family. For example, Lyapunov orbits can be evolved naturally by varying the x-coordinate of the IC.

The family must be sorted (or be loaded from an already sorted family file) before you can retrieve family members. The process will run without sorting, but the results will likely be wonky.

Exceptions
Exceptionif the sorting type is not recognized
